# Налаштування бота (GitHub Actions, безкоштовно)

## Крок 1. Створи репозиторій
1. Зайди на github.com → "New repository"
2. Назва — будь-яка, наприклад `econ-trump-bot`
3. **Обов'язково Public** (для приватних репо безкоштовний ліміт Actions — 2000 хв/міс,
   а моніторинг Трампа кожні 5 хв з'їсть його за кілька днів; публічний репо = безлімітно)
4. Створи

## Крок 2. Завантаж файли
Завантаж усі файли з цього проєкту в репозиторій, зберігаючи структуру папок
(особливо важливо: `.github/workflows/` має лишитись саме такою папкою).

Найпростіше — через веб-інтерфейс GitHub: "Add file" → "Upload files" → перетягни всі файли й папки.

## Крок 3. Додай секрети
Settings → Secrets and variables → Actions → "New repository secret":
- `TELEGRAM_BOT_TOKEN` → твій токен від @BotFather
- `TELEGRAM_CHAT_ID` → `7435765581`

## Крок 4. Дозволь боту комітити назад у репо
Settings → Actions → General → розділ "Workflow permissions" → вибери
**"Read and write permissions"** → Save.

(Без цього кроку скрипти спрацюють, але не зможуть зберігати свій стан
між запусками — і Трамп-монітор буде дублювати повідомлення.)

## Крок 5. Перевір вручну
Вкладка "Actions" у репозиторії → зліва обери workflow
("economic-calendar" або "trump-monitor") → праворуч кнопка "Run workflow" → Run.

Через 10-30 секунд перевір:
- логи запуску (натиснути на job) — там видно, чи все пройшло без помилок
- свій Telegram — мав прийти лист (для календаря — тільки якщо зараз 8:00 за Мадридом,
  інакше побачиш у логах "пропускаємо запуск", це нормально)

## Що робити, якщо щось не так

| Симптом | Що перевірити |
|---|---|
| Помилка `KeyError: TELEGRAM_BOT_TOKEN` | Секрети не додані або з помилкою в назві (мають бути рівно `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID`) |
| Помилка `git push` / `Permission denied` | Не зроблено крок 4 (Read and write permissions) |
| Календар прийшов порожній щодня | ForexFactory змінив структуру JSON-поля `date`/`impact` — напиши мені лог помилки з Actions, поправимо |
| Trump-моніторинг мовчить довго | Перевір вкладку Actions, чи workflow взагалі запускається за розкладом (іноді GitHub затримує cron на кілька хвилин — це нормально) |
| Переклад видає дивний текст або помилку | `deep-translator` (безкоштовний Google Translate) інколи блокує часті запити — якщо це стане проблемою, перейдемо на DeepL API (теж безкоштовно, 500к символів/міс, стабільніше) |

## Відомі обмеження (чесно попереджаю)

- **ForexFactory-фід неофіційний.** Це публічно використовуваний усією трейдерською спільнотою JSON, але ForexFactory може змінити чи обмежити його без попередження. Якщо це станеться — побачиш помилки в логах Actions, дай знати, підберемо альтернативу.
- **TradingEconomics не підключений** — повний доступ платний, а безкоштовний `guest` ключ дає лише демо-дані. Зараз бот працює тільки на ForexFactory, там вже покриті USD/EUR/GBP/JPY/CAD тощо. Якщо схочеш повний TE-календар — це окрема платна інтеграція, скажи, підключу.
- **Час спрацювання календаря** — не секунда в секунду 8:00, а в межах першої хвилини після початку 8-ї години (GitHub cron може на кілька хвилин запізнюватись під навантаженням).
- **Trump-моніторинг** перевіряє фід раз на 5 хвилин — це мінімальний інтервал, який підтримує GitHub Actions безкоштовно.
